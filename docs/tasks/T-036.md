# T-036: Add boot phase guidance and anti-patterns to default prompt template

- **Status**: TODO
- **Milestone**: 10 — Prompt Quality & Scalability
- **Depends**: T-030, T-043
- **PRD Reference**: §5.8

## Description

Update the default boot prompt template (created by T-030) with explicit guidance that prevents the agent from wasting tokens during the Boot phase. This is the lowest-effort, highest-impact prompt quality improvement based on log analysis.

**Requirements:**

- Add a "DO NOT re-discover tasks" instruction — the loop has already selected the task, the agent must not scan task files or check statuses
- Add a file scoping section that lists `{{task.touches}}` files (when present) as the starting point, with guidance to read those first and skip unrelated files
- Add a read budget guideline encouraging the agent to begin writing tests within ~10 tool calls
- Add an anti-patterns section covering observed failure modes:
  - After running formatters, re-read modified files (formatting may change code)
  - Write semantic test assertions, not string-matching against prompt text
  - Do not amend commits to update the SHA — the loop handles post-iteration backfill
- All guidance must be part of the externalized template at `docs/prompts/boot.md`, not hardcoded in ralph source
- All existing tests must continue to pass

## Hints

- The template uses `{{variable}}` interpolation (see T-030's prompt-template.ts)
- `{{task.touches}}` may be empty — use conditional rendering or a "not specified" fallback
- Review `.ralph-logs/` for the actual agent behaviors these anti-patterns address

## Produces

- Updated `src/ralph/templates/boot-prompt.ts` (default template content)
- Updated tests as needed
