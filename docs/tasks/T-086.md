# T-086: Remove agent instructions file generation and dependency

- **Status**: TODO
- **Milestone**: 15 — Prompt Architecture
- **Depends**: T-076
- **PRD Reference**: §3.1, §11.1, §11.5, §11.6
- **Complexity**: standard
- **Touches**: `src/ralph/commands/init.ts`, `src/ralph/core/providers/*.ts`

## Description

Ralph currently generates agent-specific instructions files (`.claude/CLAUDE.md`, `GEMINI.md`, `AGENTS.md`, `.continue/config.yaml`, `.cursor/rules/ralph.md`) during `ralph init` and previously overwrote them during `ralph update`. These files are redundant — the loop prompt is self-contained, sending everything the agent needs via the `-p` flag. They also belong to the user, not ralph — users may have their own content in these files for manual AI sessions.

This task removes the generation of agent instructions files from init and removes the `instructionsFile` capability from the provider interface.

## Changes

### init.ts

- Remove all agent instructions file generation: `generateClaudeMd()`, `generateGeminiMd()`, `generateAgentsMd()`, `generateContinueYaml()`, `generateCursorRules()`
- Remove the agent-specific branching logic that selects which file to generate
- Keep the agent selection prompt (it's still needed for `ralph.config.json`)

### Provider interface and adapters

- Remove the `instructionsFile` property from the provider interface
- Remove it from all provider adapters (claude, gemini, codex, continue, cursor)
- If any code references `instructionsFile` (e.g., for writing or reading), remove those references

### Templates

- The template files (`claude-md.ts`, `gemini-md.ts`, `agents-md.ts`, `continue-yaml.ts`, `cursor-rules.ts`) are still needed by `ralph migrate` (T-085) for template matching. Do NOT delete them — they serve as the known-template snapshots for migration. But they should no longer be imported by init.ts.

## AC

- `ralph init` no longer creates `.claude/CLAUDE.md`, `GEMINI.md`, `AGENTS.md`, `.continue/config.yaml`, or `.cursor/rules/ralph.md`
- The provider interface no longer includes `instructionsFile`
- The loop prompt remains fully self-contained (no behavior change — this is already the case)
- Template files are preserved for migration use (T-085)
- Existing tests updated, new tests verify init no longer generates agent files
- `pnpm check` passes
