# T-063: Fail-fast prompt guidance for repeated blockers

- **Status**: TODO
- **Milestone**: 13 — Loop Efficiency
- **Depends**: T-040
- **PRD Reference**: §3.2
- **Complexity**: simple

## Description

The retry context (T-040) already feeds previous failure details into the next attempt's prompt. However, the agent doesn't know it should bail early if it hits the exact same wall. Add explicit guidance to the system prompt telling the agent to stop immediately and report BLOCKED when it encounters the same blocker as a previous attempt.

## Requirements

1. **Update system prompt template** — add a section that instructs the agent: "If retry context is present and you encounter the same blocker described in the previous failure, do NOT continue trying workarounds. Instead, immediately output `[BLOCKED] <reason>` and end your turn."

2. **Parse BLOCKED signal** — the orchestrator should detect `[BLOCKED]` in agent output and treat it as a task failure with reason, saving turns and cost.

3. **Log the block** — write the BLOCKED reason to the JSONL log so it's visible in monitor and retry context.

4. **All existing tests must continue to pass.**

## Produces

- Updated system prompt template with fail-fast guidance
- Orchestrator detection of `[BLOCKED]` signal
- Tests for BLOCKED detection and logging
