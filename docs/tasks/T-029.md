# T-029: Add input sanitization for shell command arguments

- **Status**: TODO
- **Milestone**: 8 — Robustness
- **Depends**: none
- **PRD Reference**: §8.9

## Description

While the codebase uses `execFile` (array-form, no shell interpolation), user-influenced strings like task IDs, titles, and file paths are passed to shell commands without explicit validation. Add defense-in-depth sanitization.

**Requirements:**

- Audit all `execFile`/`spawn` call sites for user-influenced arguments (task IDs, titles, paths, config values)
- Add a shared validation/sanitization utility that rejects or escapes dangerous characters
- Apply the utility at all call sites where user-influenced data enters shell arguments
- Verify no call sites use shell-interpolated execution (e.g., `exec` with string commands)
- All existing tests must continue to pass

## Produces

- `src/ralph/core/sanitize.ts` (or added to an existing utility module)
- Updated call sites in `src/ralph/core/git.ts`, `src/ralph/core/process.ts`, and any other affected modules
- Tests for the sanitization utility
