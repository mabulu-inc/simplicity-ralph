# T-090: Redesign coaching as AI-powered analysis, not heuristic checks

- **Status**: TODO
- **Milestone**: 16 — Observability
- **Depends**: T-083, T-084
- **PRD Reference**: §3.12, §11.6
- **Complexity**: heavy
- **Touches**: `src/ralph/core/coach.ts`, `src/ralph/commands/review.ts`

## Description

The current `ralph review --coach` implementation is a set of shallow heuristic checks: count words in descriptions, check for AC sections, compare turn usage to limits. The output is mechanical, repetitive, and not actionable. Real problems:

1. **Ignores task ID** — `ralph review T-089 --coach` runs project-wide coaching, ignoring the specified task entirely.
2. **No role analysis** — the output contains zero information about role commentary or participation quality.
3. **Context-blind** — flags "missing AC" on 80 DONE tasks that were written before the AC convention existed. Suggests "downgrade from light to save budget" (there's nothing below light). Treats all tasks identically regardless of status, age, or project evolution.
4. **Not AI-powered** — despite being part of an AI development tool, coaching uses simple conditionals instead of leveraging the configured AI agent for intelligent analysis.

Replace the heuristic coach with an AI-powered coaching system that sends execution data to the configured AI agent and returns contextual, specific analysis.

## Changes

### coach.ts — complete rewrite

**Task-specific coaching (`ralph review T-NNN --coach`):**

- Gather the task's execution data:
  - Task file content (full markdown)
  - Log file(s) for the task — extract role commentary (`[ROLE: ...]` markers), phase durations, errors, outcome
  - Retry history (all attempts)
  - The effective prompt that was sent (or a summary of it)
- Build a coaching prompt that includes the execution data and asks the AI to analyze:
  - What went well and what could improve (specific to this task)
  - Role observation — which roles provided substantive input, which were superficial
  - Task definition quality — was the description sufficient? Were AC clear? Would hints have helped?
  - Concrete suggested changes
- Send the coaching prompt to the configured AI agent via the provider interface (same mechanism the loop uses, but with a coaching-specific prompt)
- Parse and display the AI's response

**Project-wide coaching (`ralph review --coach` without task ID):**

- Aggregate data across all tasks:
  - Task statuses, retry counts, costs, turn usage
  - Role commentary summaries (not full logs — summarize per-task)
  - Extension files that exist
  - Patterns: which milestones have high retry rates, which complexity tiers are miscategorized
- Build a project coaching prompt with the aggregated data
- Send to the AI and display the response
- The AI must account for project evolution — don't flag conventions on old tasks that predate those conventions

**Provider integration:**

- Use the existing agent provider to spawn a coaching session
- The coaching prompt should be a system prompt + user prompt (same layered approach as the loop)
- Use a short max-turns limit (e.g., 5) since coaching is a single-response analysis, not an iterative task
- Coaching should work with any configured agent, not just Claude

### review.ts

- When `--coach` is combined with a task ID, route to task-specific coaching (not project-wide)
- When `--coach` is used alone, route to project-wide coaching
- Display the AI's response as formatted text
- `--json` wraps the AI's response in a structured JSON envelope with metadata (task ID, timestamp, cost)

### Remove heuristic coach

- Remove the current heuristic analysis functions from coach.ts
- Remove tests for the heuristic checks
- The new AI-powered coach replaces them entirely

## AC

- `ralph review T-NNN --coach` analyzes that specific task's execution, not all tasks
- `ralph review --coach` (no task ID) analyzes patterns across all tasks
- Both modes send data to the configured AI agent and return its analysis
- The AI's response is contextual and specific, not generic boilerplate
- Role commentary from the execution log is included in the analysis
- DONE tasks are analyzed based on their actual execution, not flagged for missing conventions that didn't exist when they were created
- `--json` outputs structured JSON with the AI's analysis and metadata
- Works with any configured agent provider (not hardcoded to Claude)
- New tests cover prompt construction, provider integration, and output formatting (AI response content is not tested — testing the prompt and parsing, not the AI's intelligence)
- `pnpm check` passes
