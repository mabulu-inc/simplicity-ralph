## Objective

Create a README.md and a pragmatic, comprehensive GitHub Pages documentation site that helps developers understand the Ralph Methodology and maximize their use of the `@smplcty/ralph` CLI. No fluff or bloat — every page should earn its place.

## Requirements

### README.md (project root)

Create a concise, developer-focused README that:

1. **Opens with a one-liner** explaining what ralph is (stateless, PRD-driven AI development loop)
2. **Shows the value prop** — why a developer should care (automated TDD, task-driven development, multi-agent support, cost tracking)
3. **Registry setup** — explain that `@smplcty/ralph` is published to GitHub Packages and show `.npmrc` configuration:
   ```
   @smplcty:registry=https://registry.npmjs.org
   ```
   Note: requires a GitHub personal access token with `read:packages` scope.
4. **Quick start** — get running in under 60 seconds using `pnpm dlx` or `npx`:
   ```bash
   pnpm dlx @smplcty/ralph init
   # or
   npx @smplcty/ralph init
   ```
5. **Command reference table** — all 7 commands (`init`, `loop`, `monitor`, `kill`, `milestones`, `shas`, `cost`) with one-line descriptions
6. **Link to full documentation** — points to GitHub Pages site at `https://mabulu-inc.github.io/ralph/`
7. **Requirements** — Node.js 20+

### GitHub Pages documentation site (docs-site/)

Follow the same architecture as `simplicity-schema`:

#### Infrastructure

- **Framework**: Astro + @astrojs/starlight
- **Directory**: `docs-site/` at project root (separate from internal `docs/`)
- **Base path**: `/ralph`
- **Site URL**: `https://mabulu-inc.github.io`
- **Package manager**: pnpm
- **Content directory**: `docs-site/src/content/docs/`

#### Site structure and content

Organize documentation into these sections:

1. **Getting Started**
   - `introduction.md` — What is Ralph? The methodology in 2 minutes. Why stateless loops beat persistent agents. The benefit: you define tasks, ralph drives your AI agent through red/green TDD automatically.
   - `quick-start.md` — Step-by-step from zero to first completed task. Starts with `.npmrc` setup for GitHub Packages (`@smplcty:registry=https://registry.npmjs.org` + auth token with `read:packages` scope), then `pnpm dlx @smplcty/ralph init`, write a task, run `ralph loop`. Show real terminal output.
   - `configuration.md` — `ralph.config.json` fields, supported agents, file naming, quality checks.

2. **Core Concepts**
   - `methodology.md` — The Ralph Methodology explained: stateless boot, PRD-driven, task eligibility, red/green/verify cycle, quality gates. Reference `docs/RALPH-METHODOLOGY.md` for source material.
   - `tasks.md` — Task file format, status values (TODO/IN-PROGRESS/DONE/BLOCKED/SKIPPED), dependencies, metadata fields, eligibility rules, complexity tiers (Light/Standard/Heavy).
   - `prd.md` — How to write an effective PRD for ralph. Structure, inline extraction, how ralph uses it during boot.
   - `prompts.md` — Boot prompt templating, the 11 template variables, `docs/prompts/boot.md` and `docs/prompts/rules.md` customization.

3. **Commands**
   - `init.md` — Interactive project bootstrapper. What it scaffolds, agent-specific files, prompt templates.
   - `loop.md` — The main loop. How it works: boot → find eligible task → spawn agent → red/green/verify → commit → next. Flags, timeout scaling, retry context injection, codebase indexing.
   - `monitor.md` — Real-time progress dashboard. Phase tracking, last output display, activity indicators.
   - `kill.md` — Force-stop ralph and child processes. Process tree termination.
   - `milestones.md` — Generate milestone summaries with cost breakdowns.
   - `shas.md` — Backfill commit SHAs into task metadata.
   - `cost.md` — Calculate token usage and costs from JSONL logs.

4. **Agents**
   - `overview.md` — Multi-agent support. Provider abstraction pattern. Supported agents: Claude Code, Gemini CLI, Codex CLI, Continue, Cursor.
   - `claude.md` — Claude Code setup and configuration.
   - `gemini.md` — Gemini CLI setup.
   - `codex.md` — Codex CLI setup.
   - `continue.md` — Continue setup.
   - `cursor.md` — Cursor setup.

5. **Guides**
   - `writing-tasks.md` — Best practices for writing effective task files. Dependency graphs, complexity hints, keeping tasks small.
   - `customizing-prompts.md` — How to tune boot prompts and rules for your project. Template variable reference.

#### Tone and style

- Direct, declarative language. No marketing speak.
- Code-first: every concept gets a real example.
- Progressive disclosure: overview → quick start → reference → deep-dive.
- Use Starlight callout boxes (`::: tip`, `::: note`, `::: caution`) sparingly and only when genuinely useful.

- **Practical, not theoretical** — real examples, real terminal output, real task files
