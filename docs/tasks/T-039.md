## Description

Generate a lightweight codebase index at the start of each loop iteration and inject it into the boot prompt as `{{codebaseIndex}}`. This gives the agent a map of the codebase so it can surgically read only the files it needs, instead of exploring everything during Boot.

**Requirements:**

- Scan source files matching configurable glob patterns (default: `src/**/*.ts`, but should respect the project's language/structure)
- For each file, extract exported symbol names (functions, classes, interfaces, types, constants)
- Format as a compact index: one line per file with path and exported names
- Inject as the `{{codebaseIndex}}` template variable
- The index must be regenerated at the start of each iteration (files may have changed)
- Keep the index under a reasonable token budget (e.g., truncate if > 200 lines, prioritize files listed in `Touches`)
- For non-TypeScript projects, fall back to a simple file listing (path only, no export extraction)
- All existing tests must continue to pass

## Hints

- For TypeScript, a simple regex over file contents (`export (function|class|interface|type|const) (\w+)`) is sufficient — no need for a full AST parser
- Consider caching the index within an iteration (generate once, reuse if needed)
- The glob pattern could come from `ralph.config.json` or be inferred from the `language` config field

## Produces

- `src/ralph/core/codebase-index.ts`
- Updated loop orchestrator to generate the index before prompt interpolation
- Tests for index generation
