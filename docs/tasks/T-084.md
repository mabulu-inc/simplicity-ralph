## Description

After users have run ralph on several tasks, they accumulate patterns — tasks that consistently exceed their complexity tier, roles that always skip, extensions that could help but don't exist yet. The coaching feature analyzes all completed and failed tasks, role participation, and extensions to surface actionable improvement suggestions.

This runs across all tasks, not a single one. It's the difference between "what went wrong with T-042" (T-083) and "how can I get better results overall" (this task).

## Changes

### New: src/ralph/core/coach.ts

Create a coaching analysis module with three analysis areas:

**Task quality analysis:**

- Scan all TODO tasks for quality issues:
  - Missing or short Description (< 50 words)
  - No AC section
  - Missing PRD Reference
  - Missing Depends when the task's Touches overlap with files in other tasks
- Scan completed tasks for tier mismatches:
  - Tasks that used > 80% of their turn limit → suggest upgrading complexity tier
  - Tasks that used < 30% of their turn limit → suggest downgrading to save budget
  - Tasks that were retried → analyze whether Hints could have prevented the retry
- Output: list of suggestions with task ID, issue, and recommended action

**Role effectiveness analysis:**

- Parse all completed task logs for role commentary
- Identify roles that skip on > 80% of tasks → suggest disabling via `docs/prompts/roles.md`
- Identify roles whose Verify phase commentary is frequently followed by a retry → suggest refining the role's review criteria via Override
- If `docs/prompts/roles.md` defines custom roles that are never referenced in any task's `Roles` field → flag as unused
- Output: role-level suggestions

**Extension health analysis:**

- Check which extension files exist in `docs/prompts/`
- If `rules.md` is empty or contains only the default scaffold → suggest rules based on patterns (e.g., if all tasks touch files under `src/app/`, suggest a scoping rule)
- If no `roles.md` exists but roles frequently skip → suggest creating one with Disable directives
- If tasks frequently fail at Verify → suggest adding system.md extensions with project-specific quality guidance
- Output: extension-level suggestions

### review.ts

- Add `--coach` flag to the review command
- When `--coach` is used without a task ID, run the coaching analysis
- Output categorized suggestions with priority (high/medium/low) based on frequency and impact

## AC

- `ralph review --coach` analyzes all tasks and logs, outputs categorized improvement suggestions
- Task quality suggestions flag vague descriptions, missing AC, missing dependencies, and complexity tier mismatches
- Role effectiveness suggestions identify roles to disable, refine, or remove
- Extension health suggestions identify missing or underutilized extension files
- Suggestions include the specific task ID, issue description, and recommended action
- `--json` outputs structured JSON
- Handles projects with no completed tasks gracefully (reports "not enough data" rather than erroring)
- New tests cover each analysis area with fixture data
- `pnpm check` passes
