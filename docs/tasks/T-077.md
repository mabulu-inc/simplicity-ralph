## Description

Users need to customize agent roles for their project: override a built-in role's description, add project-specific roles (e.g., Compliance Officer), or disable roles that don't apply (e.g., UX/UI Designer for a headless API). PRD §9.6 defines the format and merge semantics.

## Changes

### New: src/ralph/core/roles.ts

Create a roles module that:

- Defines the built-in role list as a structured data type (name, focus, responsibility, participates phases)
- Parses `docs/prompts/roles.md` using the heading-directive format from §9.6:
  - `## Override: <RoleName>` — replaces the built-in role's description
  - `## Add: <RoleName>` — adds a new role with Focus, Responsibility, and Participates fields
  - `## Disable: <RoleName>` — removes the role
- Merges built-in roles with user customizations
- Exports the merged role list for prompt injection

### prompt-template.ts

- Import the roles module
- At prompt build time, merge built-in roles with user customizations from `docs/prompts/roles.md`
- Inject the merged role definitions into the system prompt (replacing the static role text that T-075 added)

### Task file support

- Parse an optional `Roles` field from task files in `src/ralph/core/tasks.ts`
- When `Roles` is present (comma-separated list), only those roles participate (plus Engineers, which always participate)
- Pass the active role list to the prompt builder so it can filter the role definitions and participation rules

## AC

- Built-in roles are always present when no `docs/prompts/roles.md` exists (zero-config)
- `## Override:` replaces a built-in role's description
- `## Add:` introduces a new role with participation phases

- Task files with a `Roles` field restrict participation to listed roles + Engineers
