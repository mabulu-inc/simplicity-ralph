## Description

When a task fails (timeout, non-zero exit, no commit detected) and is retried, inject context from the previous attempt into the boot prompt so the agent can avoid repeating the same mistake.

**Requirements:**

- After a failed iteration, parse the most recent log file for the failed task to extract:
  - Last phase reached (Boot, Red, Green, Verify, Commit)
  - Last error output or failure message (e.g., test failures, lint errors, timeout)
  - Files that were modified during the attempt (from tool call logs)
- Format this into a retry context block and inject as the `{{retryContext}}` template variable
- On the first attempt for a task, `{{retryContext}}` is empty
- The retry context should instruct the agent to:
  - Not repeat the same approach that failed
  - Focus on the failure point
  - Reference specific files from the previous attempt
- Cap the retry context to a reasonable size (e.g., 2000 chars) to avoid bloating the prompt
- All existing tests must continue to pass

## Hints

- Log files are JSONL at `.ralph-logs/T-NNN-*.jsonl` — use the existing log parsing patterns from `cost.ts`
- Phase detection: look for `[PHASE] Entering:` markers in text content blocks
- Error extraction: look for the last `result` type entries or stderr content
- The loop orchestrator (from T-023) is the natural place to track retry state
