## Description

Users currently create task files by hand or by copying an existing one. This is error-prone — they may miss required fields, use the wrong format, or not know which sections reach the agent. A `ralph task` command scaffolds a properly-formatted task file with the next available number, making the Task File API (§12.2) discoverable through usage.

## Changes

### New: src/ralph/commands/task.ts

- Scan `docs/tasks/` for the highest `T-NNN` number, increment by one
- Generate a task file from the built-in template (or from `docs/prompts/task-template.md` if it exists)
- Pre-fill fields from CLI flags: `--depends`, `--complexity`, `--milestone`, `--prd-ref`, `--touches`, `--roles`
- Include a guidance comment in the generated file explaining which sections reach the agent and which are excluded
- Print the created file path to stdout
- Support `--dry-run` to print to stdout without creating the file

### New: src/ralph/templates/task-scaffold.ts

- Built-in task template with:
  - All required fields (Status: TODO, Milestone, Depends, PRD Reference) with placeholder values
  - Common optional fields (Complexity, Touches, Roles) commented or with placeholder values
  - Description and AC sections with brief guidance
  - HTML comment block explaining the section inclusion/exclusion rules and `ralph show task` for verification
- Supports `{{task.number}}` and `{{task.title}}` template variables

### cli.ts

- Register the `task` command

### Extension support

- Check for `docs/prompts/task-template.md` at runtime
- If it exists, use it as the template instead of the built-in default
- The user template must include `{{task.number}}` and `{{task.title}}` placeholders (warn if missing)

## AC

- `ralph task "Title"` creates `docs/tasks/T-NNN.md` with the next available number
- The generated file has all required fields and a guidance comment about section handling
- `--depends`, `--complexity`, `--milestone`, `--prd-ref`, `--touches`, `--roles` flags pre-fill the corresponding fields
- `--dry-run` prints to stdout without creating the file
- If `docs/prompts/task-template.md` exists, it is used as the template
- If `docs/prompts/task-template.md` is missing `{{task.number}}` or `{{task.title}}`, a warning is printed
- Task number auto-detection handles gaps in numbering (e.g., T-053 missing still increments from the highest)
- New tests cover scaffolding, flag handling, auto-numbering, custom template, and dry-run
- `pnpm check` passes
