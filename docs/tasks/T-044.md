## Description

The `ralph init` interactive flow collects language, package manager, test framework, database, agent, and model — but never asks for file naming convention. The `fileNaming` field is supported by `ralph.config.json`, the rules template, and the boot prompt (`{{config.fileNaming}}`), but init doesn't populate it.

**Requirements:**

- Add a "File naming convention" prompt to the interactive init flow (after language, before package manager)
- Default to empty (no convention enforced) — user can type `kebab-case`, `snake_case`, `camelCase`, etc.
- When provided, include `fileNaming` in the returned `InitAnswers` so it flows through to `ralph.config.json`, `rules.md`, and agent instructions files
- All existing tests must continue to pass

## Produces

- Updated `src/ralph/commands/init.ts` — new prompt in `promptForAnswers()`
- Updated tests as needed
