## Description

Introduce a provider interface that abstracts the AI coding agent CLI. This decouples ralph's orchestration logic from any specific agent's CLI flags and output format.

**Requirements:**

- Define an `AgentProvider` interface with the capabilities listed in §10.1: `binary`, `buildArgs()`, `outputFormat`, `supportsMaxTurns`, `instructionsFile`, `parseOutput()`
- Define a normalized internal event format that `parseOutput()` maps agent-specific output into
- Implement the Claude Code provider as the default (extracting current hardcoded behavior)
- Register providers by name (e.g., `claude`, `gemini`, `codex`, `continue`, `cursor`)
- Update the loop orchestrator to use the provider interface instead of hardcoded `claude` CLI invocations
- Add `--agent <name>` flag to `ralph loop`
- Add `agent` field to `ralph.config.json` (default: `claude`)
- When an agent does not support `--max-turns`, rely on ralph's timeout mechanism only
