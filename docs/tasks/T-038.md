# T-038: Inline PRD section injection in boot prompt

- **Status**: TODO
- **Milestone**: 10 — Prompt Quality & Scalability
- **Depends**: T-030, T-024, T-043
- **PRD Reference**: §5.4

## Description

Instead of passing a PRD section reference (e.g., `§3.2`) and having the agent read the entire PRD to find it, ralph should extract the referenced section at prompt build time and inject the actual content into the boot prompt.

**Requirements:**

- Parse the task's `PRD Reference` field (e.g., `§3.2`, `§8.1, §8.7`)
- Extract the corresponding section(s) from `docs/PRD.md` using the Markdown AST utilities
- Inject the extracted content as the `{{task.prdContent}}` template variable
- Handle multiple references (comma-separated) by concatenating the extracted sections
- If a referenced section is not found, include a warning in the injected content (do not fail silently)
- All existing tests must continue to pass

## Hints

- Use the shared Markdown utilities from `core/markdown.ts` (consolidated in T-024) to extract sections by heading
- Section references like `§3.2` map to headings like `### 3.2` or `## 3.2` — match on the numeric prefix
- Keep extracted content trimmed to the section body (exclude sub-sections that belong to siblings)

## Produces

- `src/ralph/core/prd-extractor.ts` (or added to prompt-template.ts)
- Updated prompt template interpolation to support `{{task.prdContent}}`
- Tests for PRD section extraction and edge cases
