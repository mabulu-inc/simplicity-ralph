## Description

The tool hardcodes `origin` as the git remote and `main` as the branch. This fails for projects using `upstream`, `develop`, or custom branch naming conventions.

**Requirements:**

- Auto-detect the current branch and its tracking remote as defaults
- Allow overrides via project config (`.claude/CLAUDE.md` section) or environment variables (e.g., `RALPH_GIT_REMOTE`, `RALPH_GIT_BRANCH`)
- Update all git operations in `core/git.ts` and `commands/loop.ts` to use the configured values
- Update `scripts/ralph.sh` to respect the same configuration

## Produces

- Updated `src/ralph/core/git.ts`
- Updated `src/ralph/commands/loop.ts`
- Updated `scripts/ralph.sh`
- Tests for remote/branch detection and override

## Completion Notes

Added `detectCurrentBranch()`, `detectTrackingRemote()`, and `resolveGitTarget()` to `core/git.ts`. The `resolveGitTarget()` function auto-detects the current branch and its tracking remote, falling back to `origin` when no tracking remote is configured. Environment variables `RALPH_GIT_REMOTE` and `RALPH_GIT_BRANCH` override auto-detection. Updated `loop.ts` to use `resolveGitTarget()` instead of hardcoded `origin`/`main`. Updated `ralph.sh` with equivalent `detect_git_target` function. 11 new tests covering detection, fallback defaults, and env var overrides.
