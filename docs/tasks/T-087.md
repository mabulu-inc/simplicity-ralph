# T-087: Improve migrate template matching with historical snapshots and content extraction

- **Status**: TODO
- **Milestone**: 8 — Robustness
- **Depends**: T-085
- **PRD Reference**: §3.11
- **Complexity**: standard
- **Touches**: `src/ralph/commands/migrate.ts`, `src/ralph/templates/snapshots/`, `src/ralph/commands/loop/orchestrator.ts`, `src/ralph/commands/show.ts`

## Description

The current `ralph migrate` implementation only matches files against the current version's templates. Templates change across releases, so files generated by older ralph versions show as "unrecognized" — a status that tells the user nothing actionable. In practice, most legacy files are unmodified copies of an older template version, or older copies with a few user additions.

This task adds a library of historical template snapshots and content-aware diffing so that migrate can:

1. Recognize files generated by ANY previous ralph version, not just the current one
2. Extract user-added content from modified templates, preserving it as an extension
3. Report clear, actionable statuses instead of "unrecognized"

## Changes

### New: src/ralph/templates/snapshots/

Create a directory containing historical template snapshots extracted from git history. Each snapshot is a plain text file named by template type and version:

- `system-prompt-v0.5.0.txt`, `system-prompt-v0.6.0.txt`, `system-prompt-v0.6.2.txt`, etc.
- `boot-prompt-v0.5.0.txt`, `boot-prompt-v0.6.0.txt`, etc.
- `methodology-v0.5.0.txt`, etc.
- `claude-md-v0.5.0.txt`, `gemini-md-v0.5.0.txt`, etc.
- `prompts-readme-v0.5.0.txt`, etc.

Extract these from `git log --all -- src/ralph/templates/*.ts` — each time a template changed, that's a snapshot. Store the rendered output (not the TypeScript source), since that's what the user's file contains.

Include a script or instructions for generating new snapshots when templates change in future releases.

### migrate.ts — improved matching

Replace the current single-template exact matching with:

**Step 1: Exact match against all snapshots**

For each legacy file, normalize whitespace and compare against every historical snapshot for that file type. If any match, classify as "unmodified copy" and auto-remove.

**Step 2: Closest-match diffing for modified files**

If no exact match, find the snapshot with the highest similarity (e.g., longest common subsequence or line-level Jaccard similarity). If similarity exceeds a threshold (e.g., 60%), classify as "modified ralph template":

- Diff against the closest snapshot
- Extract lines/sections present in the user's file but not in the snapshot — these are user additions
- Display the extracted content to the user before confirming
- On confirmation, rewrite the file to contain only the extracted user content

**Step 3: Clear status for non-ralph files**

If no snapshot exceeds the similarity threshold, the file is not ralph-generated. Report as "not a ralph file" (not "unrecognized") and skip it.

### migrate.ts — improved status messages

Replace the current statuses:

| Old status              | New status                                               | Meaning                                         |
| ----------------------- | -------------------------------------------------------- | ----------------------------------------------- |
| `✓ REMOVE`              | `✓ REMOVE (unmodified v0.6.0 template)`                  | Exact match, auto-delete                        |
| `⚠ SKIP (unrecognized)` | `⚡ MODIFIED (based on v0.6.0, N lines of user content)` | Modified ralph template, show extracted content |
| `⚠ SKIP (unrecognized)` | `– SKIP (not a ralph file)`                              | Not ralph-generated, leave alone                |

### ralph show — extension highlighting

Update `formatMonitorOutput` (or the show command's display logic) to visually distinguish built-in content from user extensions in the output. Use labeled sections:

```
System prompt (built-in):
  ...

System prompt (your extensions):
  ► line 1
  ► line 2
```

### orchestrator.ts — preflight legacy file detection

During the loop's preflight step, run `migrate --dry-run` internally (call the dry-run function directly, not as a subprocess). If it finds legacy files to act on, print the plan summary and warn: `Legacy prompt files detected. Run 'ralph migrate' to clean up. Continuing with built-in templates.` The loop proceeds — this is a warning, not a blocker.

### migrate.ts — "nothing to migrate" message

When no legacy files are found, report: `No legacy files found — your project is using the built-in-first architecture. No migration needed.`

## AC

- Historical snapshots exist for every distinct template version in git history
- `ralph migrate` recognizes files from older ralph versions via snapshot matching
- Modified ralph templates are diffed against the closest snapshot, with user content extracted and displayed
- User is prompted to confirm before extraction overwrites the file
- Status messages are clear and actionable: "unmodified v0.X.0 template", "modified (N lines of user content)", "not a ralph file"
- "Unrecognized" is never shown as a status
- `ralph show` output visually distinguishes built-in content from user extensions
- Loop preflight runs migrate dry-run and warns if legacy files are detected (non-blocking)
- `ralph migrate` with no legacy files reports a helpful "nothing to migrate" message
- New tests cover multi-version matching, content extraction, similarity threshold, show highlighting, and preflight warning
