## Description

Before starting a task, the orchestrator should run the project's quality check command (`pnpm check` or equivalent) to capture any pre-existing failures. This baseline is injected into the boot prompt so the agent knows which failures it didn't cause and shouldn't waste turns fixing.

## Requirements

1. **Preflight step — once per loop invocation** — when `ralph loop` starts, run the configured `qualityCheck` command once and capture its output. This runs before the first task and is reused for every task in that loop run. It does NOT re-run between tasks.

2. **Parse failure output** — extract the list of errors/warnings from the preflight output. If the preflight passes clean, inject nothing.

3. **Inject into boot prompt** — add a "Pre-existing failures" section to the boot prompt context that lists the known failures with a note: "These failures exist before your task. Do not attempt to fix them. Only ensure you do not add new failures."

4. **Log preflight result** — write the preflight output to a `preflight.jsonl` file in `.ralph-logs/` so the monitor can show it.

5. **Timeout** — preflight should have a configurable timeout (default: 120s). If it times out, log a warning and proceed without baseline.

6. **All existing tests must continue to pass.**

## Produces

- Preflight step in the loop entry point (runs once at loop start)
- Preflight result passed through orchestrator to boot prompt
- JSONL log entry for preflight results
- Tests for preflight pass, fail, and timeout scenarios
