# T-071: Rename package from @smplcty/ralph to @smplcty/ralph

- **Status**: DONE
- **Milestone**: 15 — Rebranding
- **Depends**: none
- **PRD Reference**: §1
- **Complexity**: standard
- **Completed**: 2026-03-18 10:05

## Description

Comprehensively rename the package from `@smplcty/ralph` (published to GitHub Packages) to `@smplcty/ralph` (published to npm). Every reference across the entire codebase must be updated so the project reads as though it was always `@smplcty/ralph`. The GitHub repo remains at `mabulu-inc/simplicity-ralph`.

## Pre-Task (Interactive — Must Be Done by Maintainer)

1. **Create the `@smplcty` npm org** — go to <https://www.npmjs.com/org/create> and create the `smplcty` organization.
2. **Link the GitHub repo as a Trusted Package** — in the npm org settings, configure `@smplcty/ralph` as a trusted package linked to the `mabulu-inc/simplicity-ralph` GitHub repository. This enables tokenless publishing from GitHub Actions via OpenID Connect (OIDC).
3. **Update GitHub Pages custom domain (if applicable)** — if the docs-site will move to a new domain, configure it in repo settings.

## Scope

### package.json

- Change `"name"` from `"@smplcty/ralph"` to `"@smplcty/ralph"`
- Remove or update any `publishConfig` pointing at GitHub Packages registry
- Add `"publishConfig": { "access": "public" }` if publishing as a public package on npm

### CI/CD Workflows

  - Move permissions to top level: `contents: read`, `id-token: write` (remove `packages: write`)

  - Switch `registry-url` to `https://registry.npmjs.org`
  - Replace `pnpm publish` with `npm publish --provenance --access public` (no `NODE_AUTH_TOKEN` needed)
