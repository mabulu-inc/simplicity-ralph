# T-056: Show defaults and options in init prompts

- **Status**: DONE
- **Milestone**: 8 — Robustness
- **Depends**: none
- **PRD Reference**: §3.2
- **Commit**: d5cc5cab2b9e8a37be7515ba132b7512ee0ce3e0
- **Cost**: $0.53

## Description

`ralph init` prompts are missing visible defaults and option hints. The `prompt()` helper formats as `{question} ({default}): ` but the defaults are not being passed for several prompts, and prompts with a fixed set of valid options (like Database and AI agent) don't display those options.

**Requirements:**

1. **Pass defaults to all prompts** — Every call to `prompt()` must include the default value so the user sees it in parentheses. Currently missing for at least: File naming convention, Database, Model.

2. **Show valid options for constrained prompts** — Prompts that accept a known set of values should list them. Specifically:
   - **Database**: Show common options (e.g. `PostgreSQL`, `MySQL`, `SQLite`, `none`)
   - **AI agent**: Already shows options — no change needed
   - **File naming convention**: Show common options (e.g. `kebab-case`, `snake_case`, `camelCase`)

3. **All existing init tests must continue to pass**

## Hints

- The prompting logic is in `src/ralph/commands/init.ts` around lines 251-317
- The `prompt(rl, question, defaultValue?)` helper already supports showing defaults — the fix is mostly about passing the right default values to each call
- For option display, consider a format like `Database (PostgreSQL, MySQL, SQLite, none) [none]: `

## Produces

- Updated `src/ralph/commands/init.ts` with defaults and options visible on all prompts
- Updated tests verifying defaults are displayed

## Completion Notes

- Exported `prompt()` helper and added optional `options` parameter for displaying valid choices
- Format: `Question (opt1, opt2, opt3) [default]: ` — options in parentheses, default in brackets
