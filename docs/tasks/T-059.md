## Description

After upgrading the ralph package in an existing project, there is no way to pull in updated methodology docs or prompt templates. `ralph init` re-runs the full setup (PRD, config, T-000) and prompts about every conflict. `ralph init --prompts-only` only updates `docs/prompts/*` and skips the methodology file entirely.

Add a `ralph update` command that refreshes ralph-owned files without touching user-authored content.

## Requirements

1. **Refresh methodology** — overwrite `docs/RALPH-METHODOLOGY.md` with the latest template from the installed package. This file is ralph-owned, not user-edited.

2. **Refresh prompts** — overwrite `docs/prompts/boot.md`, `docs/prompts/system.md`, and `docs/prompts/README.md` with latest templates.

3. **Refresh agent config** — regenerate the agent instructions file (`.claude/CLAUDE.md`, `.cursor/rules/ralph.md`, `GEMINI.md`, etc.) based on the current `ralph.config.json`. This file references the methodology and project config, so it should stay in sync.

4. **Do NOT touch** — `docs/PRD.md`, `docs/tasks/*`, `ralph.config.json`, `docs/prompts/rules.md` (user-authored rules). These are user content.

5. **Show what changed** — print which files were updated vs. already up to date.

6. **Register in CLI** — add `update` to the command dispatcher in `src/ralph/cli.ts`.

7. **All existing tests must continue to pass.**

## Produces

- `src/ralph/commands/update.ts`
- Updated CLI dispatcher with `update` command
- Tests for the update command

## Hints

- Read `ralph.config.json` to determine which agent is configured (claude, cursor, gemini, etc.) so you regenerate the right agent config file
- The `writeFile` helper from `init.ts` already handles file creation — consider extracting it to a shared utility
- `docs/prompts/rules.md` is user-authored (project-specific rules) and must NOT be overwritten
