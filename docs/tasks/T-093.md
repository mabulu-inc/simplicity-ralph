## Description

A third audit of the docs-site found 13 remaining issues. The entire agents section was missed by previous fixes — all 5 agent pages plus the overview page incorrectly state that ralph generates agent instructions files. Other issues include phantom status values, a stale database config field, broken landing page links, and wrong package names.

## Changes

### agents/claude.md, agents/gemini.md, agents/codex.md, agents/continue.md, agents/cursor.md

All 5 agent pages claim ralph generates an agent instructions file during init (e.g., "Ralph generates `.claude/CLAUDE.md` during init"). This is false per PRD §11.5 — ralph does not generate or manage agent instructions files. The loop prompt is self-contained.

For each page:

- Remove the claim that ralph generates the instructions file
- Replace with a note that ralph's loop prompt is self-contained and does not depend on or manage agent-specific instructions files
- Keep the agent-specific setup instructions (install, configure) — just remove the ralph-generates-it claim

### agents/overview.md

Two issues:

1. The Supported Agents table has an "Instructions File" column listing `.claude/CLAUDE.md`, `GEMINI.md`, etc. Remove this column — it contradicts PRD §11.5.
2. The Provider Interface table lists `instructionsFile` as a capability. The PRD §11.1 explicitly states this is NOT part of the interface. Replace with `supportsSystemPrompt` per the PRD.

### getting-started/configuration.md

Line 35 lists `database` as an optional config field. The PRD does not define `database` as a config field — it was removed in T-088. Remove it.

### core-concepts/tasks.md

Lines 36-42 list `IN-PROGRESS` and `SKIPPED` as valid Status values. The actual code and PRD define only `TODO`, `DONE`, and `BLOCKED`. Remove the phantom values.

### index.mdx (landing page)

Two broken links:

1. Line 9: Hero link is `/ralph/getting-started/introduction/` but the site base is `/simplicity-ralph`. Fix to `/simplicity-ralph/getting-started/introduction/`.
2. Line 13: GitHub link points to `https://github.com/mabulu-inc/ralph` but the repo is `https://github.com/mabulu-inc/simplicity-ralph`.

### agents/gemini.md — wrong install package

Line 12: Install command says `npm install -g @anthropic-ai/gemini-cli`. Gemini CLI is a Google product. Fix the package name to the correct one (check the actual Gemini CLI installation docs).

### agents/continue.md — wrong install package

Line 12: Install command says `npm install -g @anthropic-ai/continue-cli`. Continue is not an Anthropic product. Fix the package name to the correct one.
