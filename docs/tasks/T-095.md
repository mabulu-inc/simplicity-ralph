# T-095: Add textOutputFormat to AgentProvider for coaching text output

- **Status**: TODO
- **Milestone**: 16 — Observability
- **Depends**: T-094
- **PRD Reference**: §3.12
- **Complexity**: light
- **Touches**: `src/ralph/core/agent-provider.ts`, `src/ralph/providers/claude.ts`, `src/ralph/providers/codex.ts`, `src/ralph/providers/cursor.ts`, `src/ralph/providers/continue.ts`, `src/ralph/providers/gemini.ts`, `src/ralph/commands/review.ts`, `src/ralph/__tests__/review.test.ts`

## Description

`ralph review --coach` outputs raw JSON instead of human-readable text. The root cause is in `buildSpawnAgent()` in `review.ts` — it passes `outputFormat: provider.outputFormat` when building args for the spawned agent. For the Claude provider, `outputFormat` is `['--output-format', 'stream-json']`, which makes the agent produce structured JSON events. The coaching code then dumps this raw JSON as the "analysis" text.

Hardcoding `['--output-format', 'text']` in `buildSpawnAgent` would fix Claude but break other providers (e.g. Codex uses `--json` not `--output-format`). Each provider already owns the knowledge of its own CLI flags, so it should also own the knowledge of how to request plain text output.

## Changes

### agent-provider.ts

- Add `readonly textOutputFormat: string[]` to the `AgentProvider` interface — the flags needed to get plain text output from this provider's CLI

### claude.ts, cursor.ts, continue.ts, gemini.ts

- Add `textOutputFormat: ['--output-format', 'text']` to each provider definition

### codex.ts

- Add `textOutputFormat: []` — Codex outputs text by default when `--json` is omitted

### review.ts

- In `buildSpawnAgent()`, change `outputFormat: provider.outputFormat` to `outputFormat: provider.textOutputFormat` so coaching always gets prose
- No changes needed to the `--json` / `formatCoachingJson` path — that controls ralph's output envelope, not the agent's wire format

## AC

- `ralph review T-NNN --coach` outputs human-readable coaching text, not JSON
- `ralph review --coach --json` still outputs a JSON envelope containing the coaching analysis
- `buildSpawnAgent` uses `provider.textOutputFormat` instead of `provider.outputFormat`
- All five providers define `textOutputFormat`
- Existing tests pass, new/updated tests confirm text output format in spawn args
- `pnpm check` passes
