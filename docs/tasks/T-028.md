## Description

`process.ts` uses `ps ax -o pid,command` to find ralph-related processes. This scans the entire system process table, which can inadvertently capture sensitive information (API keys, passwords) from unrelated processes' command-line arguments.

**Requirements:**

- Replace global `ps ax` scanning with a PID-scoped approach (process groups, parent-child traversal, or a library like `tree-kill`)
- Track spawned child PIDs directly rather than discovering them via the global process table
- Ensure `ralph kill` still reliably finds and terminates all ralph-spawned processes
- No sensitive data from unrelated processes should ever be read or logged
- All existing tests must continue to pass

## Completion Notes

Replaced `findProcessesByPattern` (which used `ps ax -o pid,command` to scan the global process table) with a PID file approach:

- Created `src/ralph/core/pid-file.ts` with `writePidFile`, `readPidFile`, `removePidFile` functions
- The loop orchestrator now writes its PID to `.ralph-logs/ralph.pid` on start and removes it on exit (via try/finally)
- `ralph kill` reads the PID file to find the ralph process, then uses `killProcessTree` (which uses `pgrep -P` for PID-scoped child discovery)
- `ralph monitor` uses the PID file to detect running status
- Removed `findProcessesByPattern` entirely from `process.ts` — no more global process table scanning
