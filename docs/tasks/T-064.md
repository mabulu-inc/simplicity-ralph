## Description

There is no spending limit on a task or loop run. A stuck task can burn $20+ across retries. Add a per-task cost cap and a total loop budget so ralph stops before wasting money.

## Requirements

1. **Add `maxCostPerTask` to config** — maximum USD to spend on a single task across all retries (default: $10). `ralph init` should prompt for this.

2. **Add `maxLoopBudget` to config** — maximum USD for an entire loop run (default: $100). `ralph init` should prompt for this.

3. **Track cumulative task cost** — after each iteration, sum the `total_cost_usd` from all JSONL result entries for that task. If the sum exceeds `maxCostPerTask`, mark the task BLOCKED with reason "cost limit exceeded" and move on.

4. **Track loop spend** — sum all task costs in the current loop run. If it exceeds `maxLoopBudget`, stop the loop gracefully with a summary of what was completed.

5. **Display in monitor** — show current loop spend and per-task spend in the monitor output.

6. **All existing tests must continue to pass.**

## Produces

- Updated `ralph init` prompts for budget fields
