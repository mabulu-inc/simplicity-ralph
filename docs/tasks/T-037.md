# T-037: Add `Touches` and `Hints` fields to task parser

- **Status**: TODO
- **Milestone**: 10 — Prompt Quality & Scalability
- **Depends**: T-024, T-043
- **PRD Reference**: §1, §5.1

## Description

Extend the task file parser to read two new optional fields — `Touches` (a list of file paths the task will read or modify) and `Hints` (a free-text section with implementation guidance). These are injected into the boot prompt as `{{task.touches}}` and `{{task.hints}}` template variables.

**Requirements:**

- Parse the `Touches` metadata field from task files (comma-separated inline code paths, e.g., `` `src/foo.ts`, `src/bar.ts` ``)
- Parse an optional `## Hints` section from task files (free-text content, similar to how `## Description` is parsed)
- Add `touches: string[]` and `hints: string` to the `Task` interface
- Both fields are optional — `touches` defaults to `[]`, `hints` defaults to `''`
- Wire the new fields into the prompt template interpolation engine so `{{task.touches}}` and `{{task.hints}}` are available
- All existing tests must continue to pass; add new tests for the new fields

## Hints

- Follow the existing pattern in `core/tasks.ts` — `Touches` is a metadata field like `Depends`, `Hints` is a section like `Description`
- The markdown utility from T-024 should handle section extraction

## Produces

- Updated `src/ralph/core/tasks.ts`
- Updated `src/ralph/core/prompt-template.ts` (or wherever template variables are mapped)
- Tests for parsing Touches and Hints from task files
