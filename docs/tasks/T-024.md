## Description

`core/markdown.ts` and `core/config.ts` both implement logic to extract fields and sections from Markdown ASTs. This duplicated logic violates the shared-utility principle in §8.1.

**Requirements:**

- Audit `core/config.ts` and `core/markdown.ts` for overlapping AST operations (field extraction, section reading, heading traversal)
- Move all shared Markdown AST logic into `core/markdown.ts` as a unified utility
- Refactor `core/config.ts` (and any other consumers) to call into `core/markdown.ts` instead of reimplementing AST operations
- Pure refactor — no behavior changes
- All existing tests must continue to pass

## Produces

- Updated `src/ralph/core/markdown.ts`
- Updated `src/ralph/core/config.ts`
- Updated tests as needed
