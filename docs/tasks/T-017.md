# T-017: Loop complexity scaling and boot prompt update

- **Status**: TODO
- **Milestone**: 6 — Integration & Polish
- **Depends**: T-010, T-016
- **PRD Reference**: §3.2, §5

## Description

Wire task complexity scoring into the loop command so that `--max-turns` and `--timeout` auto-scale per iteration based on the target task's complexity tier.

**Changes to `src/ralph/commands/loop.ts`:**

1. **New `-m, --max-turns` option** — override auto-scaled max turns (default: auto)
2. **Auto-scaling logic** — before each iteration, call `computeTaskComplexity()` on the target task and set turns/timeout:
   - Light: 50 turns, 600s
   - Standard: 75 turns, 900s
   - Heavy: 125 turns, 1200s
3. **CLI overrides** — explicit `-m` and `-t` flags override auto-scaled values
4. **Pass `--max-turns` to claude** — add the flag to the spawned `claude --print` command
5. **Boot prompt rule** — add bash timeout guidance: "When running test/build commands via Bash, set timeout to at least 120000ms (120 seconds). Never use 30000ms or less for test/build commands."
6. **Dry-run output** — show the computed tier, turns, and timeout in dry-run config

## Produces

- `src/ralph/commands/loop.ts` (modified)
- `src/ralph/__tests__/loop.test.ts` (modified — new tests for scaling, -m flag, boot prompt rule)
