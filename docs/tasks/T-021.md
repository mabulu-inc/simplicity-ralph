# T-021: Improve error visibility in loop and git operations

- **Status**: TODO
- **Milestone**: 8 — Robustness
- **Depends**: none
- **PRD Reference**: §8.4

## Description

The loop command and git operations silently swallow errors in try/catch blocks. A failed `git push`, network error, or authentication issue is invisible to the user — the loop just continues as if nothing happened.

**Requirements:**

- Audit all try/catch blocks in `commands/loop.ts` and `core/git.ts` (and any other modules with silent catches)
- Log all caught errors to stderr and/or to the `.ralph-logs/` iteration log
- Never silently continue after a meaningful failure — at minimum, warn the user
- Add structured error context (which operation failed, what the error was)

## Produces

- Updated `src/ralph/commands/loop.ts`
- Updated `src/ralph/core/git.ts`
- Any other modules with silent error suppression
- Tests for error logging behavior
