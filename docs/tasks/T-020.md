## Description

`killProcessTree` in `core/process.ts` sends SIGTERM/SIGKILL to a single PID without traversing child processes. If Claude spawns compilers, test runners, or other children, they become orphaned processes that continue consuming resources.

**Requirements:**

- Implement actual process tree traversal (walk child PIDs via `pgrep -P` or similar)
- Alternatively, use process group IDs (`-pid`) or a library like `tree-kill`
- Ensure all descendants are terminated before reporting success
- Add tests verifying child process cleanup
