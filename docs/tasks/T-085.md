## Description

Projects created with ralph versions prior to T-076 have prompt template copies in their workspace: `docs/prompts/boot.md`, `docs/prompts/system.md`, `docs/prompts/README.md`, and `docs/RALPH-METHODOLOGY.md`. Under the built-in-first architecture these are redundant — ralph reads from compiled code. But users may have customized them, so they can't be blindly deleted.

`ralph migrate` compares each legacy file against known built-in templates, automatically removes unmodified copies, extracts user customizations into extension files, and walks the user through confirmation.

Note: the old `ralph update` command silently overwrote agent instructions files (`.claude/CLAUDE.md`, `GEMINI.md`, etc.) without confirmation when content differed. Users who ran `ralph update` after customizing their instructions file may have lost changes. The migrate command must also handle these files.

## Changes

### New: src/ralph/commands/migrate.ts

Implement the migration command:

**Legacy files to scan:**

- `docs/prompts/boot.md` — old boot prompt template copy
- `docs/prompts/system.md` — old system prompt template copy
- `docs/prompts/README.md` — old prompt directory documentation
- `docs/RALPH-METHODOLOGY.md` — old methodology copy
- `.claude/CLAUDE.md` — agent instructions file (may contain user customizations beyond the ralph stub)
- `GEMINI.md`, `AGENTS.md`, `.continue/config.yaml`, `.cursor/rules/ralph.md` — other agent instructions files

For prompt files (`boot.md`, `system.md`, `README.md`, `RALPH-METHODOLOGY.md`), the goal is removal or conversion to extensions. For agent instructions files (`.claude/CLAUDE.md` etc.), the goal is to detect whether the user added project-specific content beyond ralph's generated stub, and preserve it.

**Template matching:**

- Maintain a set of known built-in template snapshots (current + at least one prior version). These can be stored as hashes or normalized strings. Whitespace/formatting differences should be ignored during comparison (normalize before comparing).
- For agent instructions files, the known templates are the generated stubs from `generateClaudeMd()`, `generateGeminiMd()`, etc.
- For each legacy file found, compare against all known snapshots.

**Three outcomes per file:**

1. **Exact match** — file matches a known built-in template (after normalization). Delete it automatically. Report what was removed.

2. **User modifications detected** — file resembles a known template but has additions/changes. Diff against the closest matching built-in to extract user-specific content. Write the user content to the same path (effectively stripping the built-in portion). Show the user what was extracted and ask for confirmation before writing.

3. **No match** — file doesn't resemble any known template. Leave it in place (it will function as an extension under the new architecture). Warn the user and suggest `ralph show` to verify.

**Flow:**

- Scan for all four legacy files
- Run dry-run analysis first, display the plan
- Ask for confirmation (unless `--force`)
- Execute the plan
- After completion, show the effective prompts via `ralph show system-prompt` and `ralph show boot-prompt`
- Report summary: N files removed, N files migrated to extensions, N files left as-is

**Options:**

- `--dry-run` — show the plan without modifying files
- `--force` — skip confirmation prompts
