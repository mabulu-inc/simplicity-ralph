A follow-up audit after T-091 found 3 remaining stale references in the docs-site.

## Changes

### reference/extension-api.md

- Remove `{{config.database}}` from the Template Variable Reference table. This variable was removed in T-088 and only resolves to empty string for backward compatibility. Documenting it as a current API variable misleads users into thinking it does something.

### commands/migrate.md

- Clarify the `ralph update` reference on line 12. The current text says "files that were previously copied into your project by ralph init or ralph update." Change to make it clearly historical context, e.g., "files that were copied into your project by earlier versions of ralph" — no need to name specific commands that no longer exist.

### getting-started/configuration.md

- Remove the "Agent Instructions Files" section (lines 52-62) that claims init generates `.claude/CLAUDE.md`, `GEMINI.md`, `AGENTS.md`, etc. Init does not generate agent instructions files. Replace with a brief note explaining that ralph does not manage agent instructions files — the loop prompt is self-contained (§11.5). If users want a `.claude/CLAUDE.md` for their own manual Claude usage, that's their file.
