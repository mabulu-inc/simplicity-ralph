# T-061: Max retries per task with BLOCKED status

- **Status**: TODO
- **Milestone**: 13 — Loop Efficiency
- **Depends**: none
- **PRD Reference**: §3.2
- **Complexity**: standard

## Description

When a task fails repeatedly, the loop keeps retrying it indefinitely, burning tokens on the same blocker. After a configurable number of failures (default: 3), the orchestrator should mark the task as BLOCKED in its task file and move on to the next eligible task.

## Requirements

1. **Add `maxRetries` to config** — add a `maxRetries` field to `ralph.config.json` (default: 3). `ralph init` should prompt for this value.

2. **Track retry count** — the orchestrator should count how many times a task has been attempted by counting log files matching `T-{id}-*.jsonl` in `.ralph-logs/`.

3. **BLOCKED status** — when retry count >= `maxRetries`, update the task file's Status field to `BLOCKED` and add a `Blocked reason` field with the last failure's error summary.

4. **Skip BLOCKED tasks** — the loop's task selection should skip tasks with status BLOCKED (treat like DONE for scheduling purposes).

5. **All existing tests must continue to pass.**

## Produces

- Updated orchestrator with retry counting and BLOCKED logic
- Updated task parser to recognize BLOCKED status
- Updated config parser for `maxRetries`
- Updated `ralph init` to prompt for max retries
- Tests for retry limit and BLOCKED behavior
