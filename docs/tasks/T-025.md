## Description

`cost.ts` and `shas.ts` use regex-based find-and-replace to update fields in task Markdown files. This is fragile and violates both the structured parsing principle (§8.1) and the build-vs-borrow principle (§8.7).

**Requirements:**

- Replace regex-based Markdown field updates in `cost.ts` and `shas.ts` with AST-based transformations using the shared Markdown utility from `core/markdown.ts`
- The shared utility should support updating/writing field values in Markdown files, not just reading them
- All existing tests must continue to pass — no behavior changes

## Produces

- Updated `src/ralph/commands/cost.ts`
- Updated `src/ralph/commands/shas.ts`
- Updated `src/ralph/core/markdown.ts` (write/update capabilities)
- Updated tests as needed
