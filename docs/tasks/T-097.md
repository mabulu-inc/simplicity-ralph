# T-097: Pre-create log directory in `spawnWithCapture` to prevent ENOENT crash

## Description

`ralph loop` crashes with `ENOENT` when writing task log files because `git clean -fd` in `discardUnstaged()` deletes the untracked `.ralph-logs/` directory between iterations. The directory is created once at orchestrator boot but is removed by git clean before `spawnWithCapture()` tries to open a write stream.

The fix is to `mkdirSync(dirname(logFile), { recursive: true })` in `spawnWithCapture()` immediately before `createWriteStream`, making log directory creation resilient regardless of what happens earlier in the loop.

GitHub Issue: #2
