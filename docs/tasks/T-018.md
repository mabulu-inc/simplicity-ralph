# T-018: Structured Markdown parsing for task and config files

- **Status**: DONE
- **Milestone**: 8 — Robustness
- **Depends**: none
- **PRD Reference**: §8.1
- **Completed**: 2026-03-10 22:05 (7m duration)

## Description

Replace regex-based field extraction in `core/tasks.ts` and `core/config.ts` with a proper Markdown parser (e.g., `unified`/`remark`). The current regex patterns break on minor formatting variations (e.g., `*` vs `**`, extra whitespace, trailing spaces). A structured parser makes the tool resilient to cosmetic Markdown changes.

**Scope:**

- Replace regex parsing in `parseTaskFile()` (`core/tasks.ts`) with AST-based extraction
- Replace regex parsing in `parseConfig()` (`core/config.ts`) with AST-based extraction
- All existing tests must continue to pass
- Add edge-case tests for formatting variations (single asterisks, extra spaces, missing fields)
