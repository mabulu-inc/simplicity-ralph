# T-068: `ralph retry` command to re-run tasks from scratch

- **Status**: DONE
- **Completed**: 2026-03-16 20:05
- **Milestone**: 8 — Robustness
- **Depends**: none
- **PRD Reference**: §3, §8.6
- **Complexity**: standard

## Description

Users need a way to retry a BLOCKED task as if it had never been attempted — without losing old logs. A task gets blocked after exceeding the retry limit or cost cap; `ralph retry` makes it eligible again.

`ralph retry T-NNN` resets the task file to a clean TODO state and moves existing log files to an archive subfolder so the retry counter starts fresh. Only BLOCKED tasks are eligible for reset — DONE tasks should not be reset (create a new task instead).

## Requirements

1. **BLOCKED-only** — refuse to reset tasks with any status other than BLOCKED. Print an error and exit non-zero.

2. **Task file cleanup** — set `Status` back to `TODO` and remove: `Blocked reason`. All other fields (Description, Hints, Produces, Milestone, Depends, Complexity, etc.) are preserved.

3. **Log archival** — move all `.ralph-logs/T-NNN-*.jsonl` files into `.ralph-logs/T-NNN-resets/`. If the resets folder already exists (from a previous reset), add the files alongside what's already there. Do not overwrite or delete existing archived logs.

4. **Multiple task IDs** — support `ralph retry T-005 T-006 T-007` in a single invocation.

5. **Idempotent** — resetting a task that is already TODO (with no logs) is a no-op with a message, not an error.

6. **All existing tests must continue to pass.**

## Produces

- `src/ralph/commands/retry.ts` — command entry point (argument parsing, delegates to service)
- `src/ralph/core/retry.ts` — service logic (task file cleanup, log archival, dependency check)
- CLI registration in `src/ralph/cli.ts`
- Tests for task cleanup, log archival (including multiple resets), downstream warnings, multi-task invocation, and idempotency
