# T-041: Layered prompt architecture with system prompt separation

- **Status**: TODO
- **Milestone**: 10 — Prompt Quality & Scalability
- **Depends**: T-030, T-031, T-043
- **PRD Reference**: §5.7

## Description

Split the boot prompt into stable and dynamic layers to maximize API prompt caching. Stable content (methodology, tool rules, project config) should be passed as the system prompt where the agent supports it, while task-specific content remains in the user prompt.

**Requirements:**

- Split the externalized boot prompt template into two files:
  - `docs/prompts/system.md` — methodology rules, TDD workflow, tool usage, quality gates, project config (stable across iterations)
  - `docs/prompts/boot.md` — task-specific content: task description, PRD content, codebase index, touches, hints, retry context (changes per iteration)
- Update the prompt template engine to load and interpolate both templates
- Update the agent provider interface (from T-031) to support passing a system prompt separately:
  - Add `supportsSystemPrompt: boolean` and `systemPromptFlag: string` to the provider interface
  - Claude Code: `--system-prompt` flag
  - Other providers: concatenate system + boot into a single prompt if system prompt is not supported
- Update `ralph init` to generate both template files
- All existing tests must be updated

## Hints

- The system prompt contains: phase logging instructions, workflow steps, quality gates, tool usage rules, bash timeout rules, commit format — everything that doesn't change between tasks
- The user/boot prompt contains: current task info, PRD content, codebase index, file scoping, retry context — everything task-specific
- For Claude Code, `--system-prompt` is passed as a separate CLI argument

## Produces

- `src/ralph/templates/system-prompt.ts` — default system prompt template content
- Updated `src/ralph/templates/boot-prompt.ts` — now task-specific only
- Updated `src/ralph/core/prompt-template.ts` — handles two templates
- Updated agent provider interface and Claude provider
- Updated `src/ralph/commands/init.ts`
- Tests for layered prompt loading and provider integration
