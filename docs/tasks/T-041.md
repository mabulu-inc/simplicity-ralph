## Description

Split the boot prompt into stable and dynamic layers to maximize API prompt caching. Stable content (methodology, tool rules, quality gates) is project-independent and should be passed as the system prompt where the agent supports it. Task-specific and project-specific content remains in the user prompt.

**Requirements:**

- Split the externalized boot prompt template into two files:
  - `docs/prompts/system.md` — project-independent methodology rules: phase logging, TDD workflow, tool usage rules, bash timeout rules, quality gates, commit format, "complete ONE task then STOP". No `{{config.*}}` or `{{task.*}}` variables — this content is the same for every ralph project.
  - `docs/prompts/boot.md` — project-specific and task-specific content: config values, project rules, task description, PRD content, codebase index, touches, hints, retry context. Contains all `{{config.*}}`, `{{task.*}}`, `{{project.rules}}` variables.
- Update the prompt template engine to load and interpolate both templates
- **Graceful fallback**: if `docs/prompts/system.md` does not exist, concatenate all content into the user prompt (same as current behavior). This ensures backward compatibility with projects that haven't re-run init after upgrading.
- Update the agent provider interface (from T-031) to support passing a system prompt separately:
  - Add `supportsSystemPrompt: boolean` and `systemPromptFlag: string` to the provider interface
  - Claude Code: `--system-prompt` flag
  - Other providers: concatenate system + boot into a single prompt if system prompt is not supported
- Update `ralph init` to generate both template files
- All existing tests must be updated

## Hints

- The system prompt is project-independent — it contains no template variables, just ralph's methodology rules
- The user/boot prompt is where all project and task specifics live
- For Claude Code, `--system-prompt` is passed as a separate CLI argument
- The graceful fallback is critical for the self-evolution loop — `dist/` may be rebuilt mid-run before `system.md` exists
