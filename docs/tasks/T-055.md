# T-055: Preserve planning artifacts during clean slate

- **Status**: DONE
- **Completed**: 2026-03-15 14:52 (5m duration)
- **Milestone**: 11 — Post-Iteration & Housekeeping
- **Depends**: none
- **PRD Reference**: §3.2

## Description

The loop's "clean slate" step (step 3) discards all unstaged changes before each iteration to prevent half-finished code from a crashed iteration from poisoning the next one. However, this also discards human-authored planning artifacts — new task files, PRD edits, prompt changes, and config updates — that the user may have queued between iterations without committing.

**Requirements:**

1. **Selective clean slate** — Instead of discarding all unstaged changes, only discard changes outside of protected paths. Protected paths:
   - `docs/tasks/`
   - `docs/PRD.md`
   - `docs/prompts/`
   - `docs/RALPH-METHODOLOGY.md`
   - `ralph.config.json`

2. **Untracked files in protected paths should be preserved** — New task files (untracked) must not be removed by git clean or equivalent operations.

3. **Unstaged changes in all other paths are still discarded** — Source code, test files, config files outside the protected list, build artifacts, etc. should continue to be cleaned as before.

4. **All existing tests must continue to pass** — Add tests verifying that protected paths survive the clean slate while non-protected paths are discarded.

## Hints

- The clean slate logic is in the loop orchestrator. Look at how `git checkout -- .` or `git clean` is currently invoked and make it path-aware.
- `git checkout -- .` can be replaced with `git checkout -- <specific paths>` excluding protected ones, or use `git diff --name-only` to get changed files, filter out protected paths, and restore only the rest.
- For untracked files, `git clean` can accept path exclusions via `-e` patterns.

## Produces

- Updated loop orchestrator with selective clean slate
- Tests verifying protected paths survive and non-protected paths are discarded

## Completion Notes

Implemented selective clean slate in `core/git.ts` by replacing the blanket `git checkout -- .` with path-aware logic: `git diff --name-only` to find modified tracked files, filter out protected paths, and restore only non-protected files. Untracked files are cleaned via `git clean -fd` with `-e` exclusion patterns for protected paths. The `LoopGitService` passes the five protected paths (docs/tasks/, docs/PRD.md, docs/prompts/, docs/RALPH-METHODOLOGY.md, ralph.config.json) to the updated function. Added integration tests verifying modified and untracked files in protected paths survive while non-protected paths are discarded.
