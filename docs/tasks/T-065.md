# T-065: Show per-invocation progress alongside total progress in monitor

- **Status**: TODO
- **Milestone**: 13 — Loop Efficiency
- **Depends**: none
- **PRD Reference**: §3.2
- **Complexity**: simple

## Description

The monitor shows overall progress (e.g. 70/89, 79%) but when a loop starts with most tasks already done, the percentage doesn't reflect the current run's progress. Add per-invocation stats so the user can see both the big picture and how the current loop run is doing.

## Requirements

1. **Record starting state** — when `ralph loop` starts, record how many tasks were DONE at that point (e.g. write to `.ralph-logs/loop-start.json` with `{ "doneAtStart": 70, "total": 89, "startedAt": "..." }`).

2. **Monitor shows both** — display two lines:
   - `Progress: [████░░░░] 72/89 (81%)` — overall, same as today
   - `This run: 2/19 tasks completed` — tasks completed since this loop invocation started

3. **Reset on new loop invocation** — each `ralph loop` invocation overwrites the start snapshot so the per-run counter resets.

4. **All existing tests must continue to pass.**

## Produces

- Loop start snapshot written by orchestrator
- Monitor reads snapshot and displays per-invocation stats
- Tests for per-invocation progress calculation
