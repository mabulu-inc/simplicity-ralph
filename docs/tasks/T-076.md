- **Status**: DONE
- **Completed**: 2026-03-19

Duplicate detection skips extensions that match built-in content.

- **PRD Reference**: §5, §5.1, §5.2, §5.3, §5.4, §5.8
- **Complexity**: heavy
- **Touches**: `src/ralph/core/prompt-template.ts`, `src/ralph/commands/loop/orchestrator.ts`, `src/ralph/commands/init.ts`

## Description

Ralph currently copies its prompt templates into the user's project (`docs/prompts/boot.md`, `docs/prompts/system.md`) and reads from those copies at runtime. This means built-in improvements (like the agent roles added in T-075) don't reach users unless they run `ralph update`, which overwrites their customizations. This is a fundamental design flaw: users must choose between "latest ralph" and "my customizations."

The fix: ralph's code becomes the runtime source of truth for all built-in content. User project files in `docs/prompts/` become **extensions** that are appended to (not replacements for) the built-in content. If no extension files exist, ralph works with zero configuration.

## Changes

### prompt-template.ts

Refactor `loadLayeredPrompt()` and `loadAndInterpolate()`:

- Import `defaultSystemPromptTemplate()` and `defaultBootPromptTemplate()` from templates
- Always start with the built-in template content from ralph's compiled code
- Check for user extension files (`docs/prompts/system.md`, `docs/prompts/boot.md`) — if they exist, append their content after a `--- Project Extensions ---` separator
- Apply template variable interpolation to both built-in and extension content
- Remove any logic that treats user files as the primary source

### orchestrator.ts

- Update prompt building to use the refactored `loadLayeredPrompt()` (should require minimal changes if the interface is preserved)
- Remove the preflight check for `docs/prompts/boot.md` existence — it is no longer required

### init.ts

- Stop generating `docs/prompts/boot.md` and `docs/prompts/system.md` during `ralph init`
- Stop generating `docs/prompts/README.md` and `docs/RALPH-METHODOLOGY.md`
- Continue generating `docs/prompts/rules.md` (this is user-authored content)
- Remove references to the update command's template refreshing

### Migration

- If an existing `docs/prompts/system.md` or `boot.md` matches a known built-in template (exact match or close match by stripping whitespace), it can be safely ignored at runtime since it duplicates the built-in. Log a one-time notice suggesting the user delete the redundant file or convert it to an extensions-only file.
- If the file contains user modifications (doesn't match built-in), treat the entire file as extension content and append it. This preserves all existing customizations.

## AC

- `loadLayeredPrompt()` uses built-in templates from ralph's compiled code as the base
- User extension files in `docs/prompts/` are appended to built-in content when they exist
- If no extension files exist, ralph works identically to before (zero-config)
- Template variable interpolation works in both built-in and extension content
- `ralph init` no longer creates `docs/prompts/boot.md`, `system.md`, `README.md`, or `docs/RALPH-METHODOLOGY.md`
- `ralph init` still creates `docs/prompts/rules.md`
- Existing projects with old-style full-copy prompt files continue to work (content is treated as extension)
- The `ralph update` command is removed or stubbed with a message explaining it is no longer needed
- Existing tests updated, new tests cover built-in + extension merging
- `pnpm check` passes
