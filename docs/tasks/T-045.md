# T-045: Make ralph init idempotent with smart defaults and diff-based overwrite

- **Status**: TODO
- **Milestone**: 10 — Prompt Quality & Scalability
- **Depends**: T-043
- **PRD Reference**: §3.1

## Description

Running `ralph init` on an existing project forces the user to re-enter every setting and blindly overwrites (or skips) all files. This makes it painful to regenerate a single file after a template upgrade.

**Requirements:**

- On startup, if `ralph.config.json` exists, load it and use its values as defaults for every interactive prompt (project name falls back to `package.json` `name` field or directory name)
- After collecting answers, for each file init would generate:
  1. If the file does not exist → create it (current behavior)
  2. If the file exists and its contents match what would be generated → skip silently
  3. If the file exists and its contents differ → show a summary of what changed and prompt the user to overwrite or skip
- The diff check should be content-based (compare generated string to existing file), not timestamp-based
- Remove the global `overwrite` boolean — each file is handled individually based on the diff
- Report created, skipped (unchanged), and overwritten files in the summary
- All existing tests must continue to pass

## Hints

- `readConfig` already parses `ralph.config.json` — reuse it to seed defaults
- For the diff prompt, a simple "File X has changed. Overwrite? (y/N)" is sufficient — no need to show a full diff
- The `writeFile` helper already has the skip-if-exists logic — extend it with the diff check
- Consider extracting a `loadExistingDefaults()` helper that returns partial `InitAnswers`

## Produces

- Updated `src/ralph/commands/init.ts` — default loading, per-file diff check, interactive overwrite prompt
- Updated tests for init behavior with existing config and files
