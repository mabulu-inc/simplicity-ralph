## Description

Running `ralph init` on an existing project forces the user to re-enter every setting and blindly overwrites (or skips) all files. This makes it painful to regenerate a single file after a template upgrade.

**Requirements:**

- On startup, if `ralph.config.json` exists, load it and use its values as defaults for every interactive prompt (project name falls back to `package.json` `name` field or directory name)
- After collecting answers, for each file init would generate:
  1. If the file does not exist ŌåÆ create it (current behavior)
  2. If the file exists and its contents match what would be generated ŌåÆ skip silently
  3. If the file exists and its contents differ ŌåÆ show a summary of what changed and prompt the user to overwrite or skip
- The diff check should be content-based (compare generated string to existing file), not timestamp-based
- Remove the global `overwrite` boolean ŌĆö each file is handled individually based on the diff
- Report created, skipped (unchanged), and overwritten files in the summary
- All existing tests must continue to pass
