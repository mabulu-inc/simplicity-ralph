## Description

The `loop.ts` command is a monolith that handles task scanning, git operations, process orchestration, and prompt generation all in a single `run()` function. Decompose it into smaller, focused services with clear single responsibilities.

**Requirements:**

- Extract a **LoopOrchestrator** class/module that owns the iteration lifecycle (pre-flight checks, task discovery, launching Claude, post-iteration handling) and delegates to specialized services
- Extract a **PromptGenerator** service that encapsulates `generateBootPrompt()` and any future prompt construction logic
- Extract a **GitService** (loop-specific) that wraps the git operations performed during loop iterations (discard unstaged, HEAD SHA detection, push-after-commit)
- The existing `run()` function in `loop.ts` should become a thin entry point that wires the orchestrator and its dependencies together
- Each extracted service must be independently unit-testable
- All existing tests must continue to pass — this is a pure refactor with no behavior changes
- Maintain the existing public API (`run`, `parseLoopOptions`, `LoopOptions`, etc.) so CLI entry point is unaffected
