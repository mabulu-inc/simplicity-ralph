## Description

Users currently have no way to understand what happened during a task's execution without manually reading raw JSONL log files. When a task fails or gets blocked, diagnosis is guesswork. This command provides structured post-execution analysis with role interplay visualization and actionable failure diagnosis.

## Changes

### New: src/ralph/core/log-analyzer.ts

Create a log analysis module that parses JSONL log files and extracts structured data:

- **Phase timeline** — extract `[PHASE] Entering: ...` markers with timestamps, compute per-phase durations
- **Role commentary** — extract all `[ROLE: ...]` markers, organized by phase. For each role entry: the role name, the phase it appeared in, the full commentary text, and the timestamp
- **Key decisions** — identify significant moments: test strategy decisions (SDET during Red), architectural choices (Architect during Boot), security flags (AppSec), TDD compliance verdict (SDET during Verify), code review outcome (Tech Lead during Verify)
- **Failure signals** — extract blocked signals, error messages, max turns exhaustion, timeout, non-zero exit codes
- **Cost/turns** — extract from the result entry: token usage, cost, turns used, stop reason
- **Files changed** — extract file paths from Edit/Write tool calls

This module should work with the existing log format and be usable by both `ralph review` and potentially `ralph monitor` in the future.

### New: src/ralph/commands/review.ts

Implement `ralph review T-NNN` with two modes:

**Default mode (timeline):**

- Find all log files for the task (`.ralph-logs/T-NNN-*.jsonl`) — there may be multiple from retries
- For each attempt, display:
  - Attempt number and timestamp
  - Status: success (committed), failed (with reason), or blocked
  - Phase timeline with durations
  - Role commentary organized by phase — show each role's `[ROLE: ...]` output under the phase where it appeared. This is the primary visualization of role interplay.
  - Cost, turns used vs. limit
  - Files changed (if committed)
- For DONE tasks, show the final successful attempt prominently with all prior attempts as context
- For BLOCKED tasks, show all attempts with the blocking reason

**Diagnose mode (`--diagnose`):**

- Classify the failure: timeout, max_turns, quality_check, blocked_by_agent, no_commit, role_rejection
- Extract root cause: last error, failure phase, context leading up to failure
- If a role flagged an issue at a gate phase, surface that commentary as the likely cause
- Generate recommendations based on failure type:
  - Timeout → "Consider increasing complexity tier from standard to heavy, or split this task"
  - Max turns → "Add a Hints section with implementation guidance to reduce exploration"
  - Quality check → "The verify phase failed on: [specific check]. Fix: [suggestion]"
  - Role rejection → "The [role] flagged: [commentary]. Consider: [suggestion]"
  - Vague description → "The Description section is [N] words with no AC section. Add acceptance criteria."

## AC

- `ralph review T-NNN` displays a structured timeline with phase durations, role commentary per phase, cost/turns, and files changed
- Role commentary is organized by phase, showing the full `[ROLE: ...]` text for each participating role
- Multiple attempts (retries) are shown in sequence
- `ralph review T-NNN --diagnose` classifies the failure and provides actionable recommendations
- `--diagnose` surfaces role gate rejections as likely causes when applicable
- `--json` outputs structured JSON for both modes
- `--verbose` shows full commentary text (default shows abbreviated)
- Handles edge cases: task with no logs, task still running, task never attempted
