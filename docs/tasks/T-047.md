# T-047: Add current phase timing and last log line to monitor

- **Status**: TODO
- **Milestone**: 11 — Post-Iteration & Housekeeping
- **Depends**: T-023
- **PRD Reference**: §3.3

## Description

The monitor dashboard shows which phases have been reached but not _when_ the current phase started. It also doesn't show the agent's latest output, leaving users blind to what's happening between refreshes.

**Requirements:**

- Parse the latest log file to extract the timestamp of the most recent `[PHASE] Entering:` marker and display elapsed time since that phase started (e.g., "Current phase: Green (2m 15s ago)")
- Extract the last meaningful text content line from the log file (the most recent assistant text block, not tool call JSON) and display it truncated to terminal width
- Add `currentPhaseStarted: Date | null` and `lastLogLine: string | null` to the `MonitorData` interface
- Update `formatMonitorOutput()` to render both new fields
- The log file may be large and actively written — read efficiently (e.g., read the last N bytes rather than the entire file)
- All existing tests must continue to pass

## Hints

- Log entries are JSONL — each line is a JSON object. Phase markers appear in `content` blocks with `type: "text"` containing `[PHASE] Entering: <name>`
- For the last log line, look for the last entry with a text content block from the assistant role — extract the text, strip markdown/formatting, and truncate
- Use `process.stdout.columns` (or a default of 80) for terminal width
- For efficient tail reading of large files, consider reading the last ~8KB of the file rather than the whole thing
- Timestamps can be derived from the JSONL entry's position in the file relative to the file's modification time, or from embedded timestamps if present

## Produces

- Updated `src/ralph/commands/monitor.ts`
- Tests for phase timing extraction and last log line parsing
