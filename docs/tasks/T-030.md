# T-030: Externalize boot prompt as a user-editable template

- **Status**: DONE
- **Milestone**: 9 — Agent Abstraction
- **Depends**: none
- **PRD Reference**: §5
- **Completed**: 2026-03-11 08:15 (15m duration)

## Description

The boot prompt is currently hardcoded in `generateBootPrompt()` inside `loop.ts`. Externalize it as a Markdown template that `ralph init` drops into the user's project at `docs/prompts/boot.md`. The loop reads and interpolates this template at runtime.

**Requirements:**

- Create a default boot prompt template with `{{variable}}` interpolation syntax (see §5.1 for variable list)
- `ralph init` writes the default template to `docs/prompts/boot.md`
- The loop loads the template from `docs/prompts/boot.md`, interpolates variables, and passes the result as the prompt
- If the template file is missing, fail with a clear error (not a silent fallback)
- Remove the hardcoded `generateBootPrompt()` function
- All existing tests must be updated to reflect the new template-based approach

## Produces

- `src/ralph/core/prompt-template.ts` — template loader and interpolation engine
- `src/ralph/templates/boot-prompt.ts` — default template content for init
- Updated `src/ralph/commands/init.ts`
- Updated `src/ralph/commands/loop.ts`
- Tests for template loading and interpolation
